Anlisis de Sentimientos en Reseas de Aplicaciones de la Google Play Store

Este proyecto utiliza procesamiento de lenguaje natural (PLN) para realizar un anlisis de sentimientos a las reseas de las aplicaciones de Google Play Store. Bajo esta idea, se ha desarrollado un modelo de clasificacin que puede predecir si una resea es positiva, neutral o negativa.

Descripcin del Proyecto

El proyecto consta de los siguientes componentes:

Dataset: Se utiliza un conjunto de datos que contiene reseas de aplicaciones de la Google Play Store, junto con sus calificaciones y otros detalles, extraido de Kaggle.

Preprocesamiento de Datos: Se realizan tareas de limpieza y preprocesamiento en el texto de las reseas, que incluyen la eliminacin de caracteres especiales, tokenizacin, eliminacin de stopwords y lematizacin.

Entrenamiento del Modelo: Se entrena un modelo de clasificacin utilizando algoritmos de aprendizaje automtico como Naive Bayes.

Despliegue de la API: Se crea una API web utilizando Flask que permite enviar reseas y recibir la prediccin de sentimiento correspondiente.

Preprocesamiento de texto

La seccin de preprocesamiento de texto es de los pasos ms importantes para realizar este anlisis, ya que asegura que todos los textos estn listos para entrar al modelo.

Primero, se convierte el texto a minsculas y se eliminan cualquier carcter no alfabtico, como los emojis.

Despus, se tokenizan, es decir, se separan individualmente para mejorar la comprensin.

Luego, se retiran aquelas palabras que no aportan mucho significado al anlisis, lo que llamamos las palabras vacas o "stopwords".

Despus de eso, se preparan las palabras para estar casi listas para entrar al modelo, como cuando se cortan los ingredientes en la cocina, lo que se conoce como lematizar las palabras, donde se reducen las palabras a su forma base o raz.

Finalmente, se aplica la funcin de preprocesamiento al texto, donde se prepara el texto de la mejor manera posible para que pueda ser utilizado en el anlisis de sentimientos.

Entrenamiento del modelo

El entrenamiento de un modelo es como la construccin de un puente.

Primero, se recolectan los materiales necesarios, que en este caso son los datos preprocesados y etiquetados.

Luego, se dividen los datos en un conjunto de entrenamiento y otro de prueba. Esto es como disear las maquetas del puente antes de comenzar la construccin real.

Despus, se transforman los datos en una forma que nuestro modelo pueda entender. En este caso se utilizan tcnicas como TF-IDF para convertir el texto en nmeros, es decir, en un idioma que la mquina pueda entender.

A continuacin, se selecciona el algoritmo de aprendizaje, que es como elegir el tipo de construccin del puente. En este caso se escogi un algoritmo de Naive-Bayes, que se conoce por su efectividad en conjuntos de datos pequeos y medianos.

Se entrena el modelo con los datos de entrenamiento, similar a comenzar la construccin del puente segn los planos diseados, utilizando los datos de entrenamiento para ajustar los parmetros del modelo y mejorar su capacidad para predecir con precisin.

Una vez que el modelo est entrenado, se valida con datos de prueba que no ha visto antes. Es como probar la resistencia y la estabilidad del puente con pruebas de carga, ya que el puente no debe caerse cuando pasen vehculos.

Finalmente, se evala el rendimiento del modelo utilizando mtricas, que es como inspeccionar el puente terminado para asegurarse de que cumpla con los estndares de seguridad y calidad establecidos, que se pueden ver a continuacin:

Uso de la API

Para utilizar la API, enva una solicitud POST a la ruta https://rubiog.pythonanywhere.com/analiza_sentimiento en un API Tester como Postman con el siguiente formato JSON en ingls:

{
  "review": "Tu resea o comentario."
}

La API devolver una respuesta con el sentimiento predicho y su probabilidad asociada, por ejemplo:

{
  "Sentimiento": "Positive",
  "Probabilidad": "80.00"
}

Esto debe hacer en ingls, ya que es el idioma en que estaban las reseas.
